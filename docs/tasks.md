## 全体依存関係

```
Phase 0 (環境構築)
   ↓
Phase 1 (画面モック)
   ↓
Phase 2 (Firebase CRUD) ─→ Phase 3 (認証) ─→ Phase 4 (画像・絵文字)
   ↓                          ↓
Phase 5 (検索・無限スクロール) ←───────────────────┘
   ↓
Phase 6 (UI ブラッシュアップ)
   ↓
Phase 7 (リリース準備)
```

- Phase 1 は Phase 2〜5 の前提（モックがあれば各機能を段階的に実装できる）
- Phase 2, 3, 4 は並行可能（ただし認証が先だと Security Rules の設定が楽）
- Phase 5 は Phase 2 の完了が前提（検索対象は Firestore のデータ）
- Phase 6, 7 は全機能実装後に実施
