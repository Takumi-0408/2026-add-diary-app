# 日記アプリ 仕様書

## 1. プロジェクト概要

### 1.1 目的
ユーザーが日常の出来事や思い出を手軽に記録・振り返ることができる日記アプリを開発する。外部サーバーにデータを保存することで、端末紛失時や機種変更時にも日記が失われないようにする。

### 1.2 対象プラットフォーム
- iOS / Android（React Native）
- ファーストリリースはモバイルアプリのみ。Web 版は将来対応候補。

### 1.3 ターゲットユーザー
- 日記をつけたいが続かない人
- プライベートな記録を安心して保存したい人
- シンプルで温かみのあるUIを好む人

## 2. 技術スタック

### 2.1 フロントエンド
| 項目 | 採用技術 | 理由 |
|------|---------|------|
| フレームワーク | React Native（Expo 推奨） | クロスプラットフォーム開発、ホットリロード、豊富なエコシステム |
| 言語 | TypeScript | 型安全性、開発効率の向上 |
| ナビゲーション | React Navigation | 業界標準、スタック・タブ・モーダルに対応 |
| 状態管理 | React Context + useReducer（軽量）または Zustand | 小規模アプリには過剰な導入を避ける。必要に応じて追加 |
| アイコン | react-native-emoji-picker（絵文字選択） | 温かみのある表現、導入が容易 |
| 画像選択 | expo-image-picker（ギャラリー） | Expo との親和性、シンプルなAPI |

### 2.2 バックエンド
| 項目 | 採用技術 | 理由 |
|------|---------|------|
| BaaS | Firebase | 認証・DB・ストレージが1SDKで完結。Expo との連携実績豊富 |
| データベース | Cloud Firestore | リアルタイム同期、オフライン対応、スケーラブル |
| 認証 | Firebase Authentication | メール/パスワード + Google + Apple サインイン対応 |
| ストレージ | Firebase Cloud Storage | 画像アップロードに使用 |
| 画像最適化 | Firebase Extensions（Generate Thumbnail） | サムネイル自動生成（将来対応） |

## 3. プロジェクト構成 / 開発方針

### 3.1 ディレクトリ構成（案）
```
src/
├── app/              # App.tsx, ナビゲーション設定
├── features/
│   ├── auth/         # 認証関連（ログイン・サインアップ）
│   ├── diary/        # 日記一覧・作成・詳細・編集・検索
│   └── common/       # 共有コンポーネント
├── services/
│   ├── firebase.ts   # Firebase 初期化・設定
│   ├── diaryService.ts  # 日記CRUD操作
│   └── authService.ts   # 認証操作
├── types/            # TypeScript 型定義
└── utils/            # 日付フォーマット等のユーティリティ
```

### 3.2 開発方針
- **コンポーネント駆動開発**: 画面は小さなコンポーネントの組み合わせで構築
- **段階的実装**: Phase 1（画面モック）→ Phase 2（Firebase連携）→ Phase 3（認証・画像）→ Phase 4（検索・無限スクロール）
- **テスト**: 手動テストを前提とするが、ユーティリティ関数は単体テストを推奨
- **デザイン一貫性**: カラートークン・フォント・スペーシングを定めたデザインシステムを事前に定義する

### 3.3 デザイン方針
- **温かみのある手書き風**のテイストを採用
- 角丸・余白を多めに取り、親しみやすい印象に
- 手書き風フォント（例：Zen Kurenaido, Kosugi Maru 等）を使用
- パステルカラーを基調とした優しい配色
- ダークモード対応（将来対応）

## 4. 画面仕様

### 4.1 日記一覧（ホーム）画面
| 項目 | 仕様 |
|------|------|
| 役割 | 自分の日記を時系列で一覧表示するトップ画面 |
| ナビゲーション | 下部タブまたはヘッダー左に検索アイコン、ヘッダー右に作成ボタン |
| 一覧表示 | 日記カードを日付降順で表示。各カードにアイコン・タイトル・日付・本文の抜粋を表示 |
| 無限スクロール | 初回読み込み 10 件、スクロールに応じて追加 10 件ずつフェッチ。Firestore のページネーション（cursor-based）を使用 |
| 検索バー | 画面上部に簡易検索バーを常設。タイトルまたは本文の部分一致検索。入力ごとに絞り込み |
| 空状態 | 日記が1件もない場合、「はじめての日記を書こう」というメッセージ + 作成ボタンを表示 |
| ローディング | スケルトンスクリーンまたはインジケーターで読み込み中を表示 |
| プルダウンリフレッシュ | 最新データを再取得 |

### 4.2 日記作成画面
| 項目 | 仕様 |
|------|------|
| 役割 | 新しい日記を書く |
| 遷移 | ホームの作成ボタンからスタック遷移 |
| アイコン | 絵文字ピッカーから1つ選択（必須）。カテゴリ別またはグリッド表示 |
| タイトル | テキスト入力（必須、最大50文字） |
| 本文 | テキストエリア（必須、最大5000文字） |
| 日付 | デフォルトで現在日時。カレンダーから変更可能（必須） |
| 画像 | ギャラリーから1枚選択可能（任意）。選択後はプレビュー表示、削除も可能 |
| 保存ボタン | 全必須項目が埋まったら有効化。保存後は Cloud Firestore に書き込み、一覧画面に戻る |
| キャンセル | 未保存で戻る場合、「保存されていない変更があります」アラートを表示 |

### 4.3 日記詳細画面
| 項目 | 仕様 |
|------|------|
| 役割 | 日記の全文を読む |
| 遷移 | 一覧のカードタップでスタック遷移 |
| 表示項目 | アイコン（大表示）・タイトル・日付・本文・画像 |
| 編集ボタン | ヘッダーまたは画面下部に編集アイコン |
| 削除ボタン | 確認アラート後、Firestore から削除し一覧に戻る |
| 画像 | タップで拡大表示（Lightbox 風） |

### 4.4 日記編集画面
| 項目 | 仕様 |
|------|------|
| 役割 | 既存の日記を編集する |
| 遷移 | 詳細画面の編集ボタンからスタック遷移 |
| 編集項目 | アイコン・タイトル・本文・日付・画像（すべて作成画面と同様） |
| 初期値 | 既存データをフォームにセットして表示 |
| 保存ボタン | Firestore の該当ドキュメントを更新し、詳細画面に戻る |
| キャンセル | 変更がある場合のみ確認アラート |

### 4.5 日記検索画面
| 項目 | 仕様 |
|------|------|
| 役割 | 詳細な条件で日記を検索する |
| 遷移 | ホームの検索アイコンからスタック遷移、またはタブで独立表示 |
| 検索条件 | フリーワード（タイトル・本文）・日付範囲・アイコン絵文字 |
| 結果表示 | 一覧画面と同様のカード形式。ヒット件数を表示 |
| 空結果 | 「該当する日記が見つかりませんでした」メッセージ |
| 検索履歴 | 直近の検索ワードを表示（後方互換、将来対応） |

## 5. データモデル

### 5.1 Firestore コレクション構成
```
users/{userId}               # ユーザープロフィール
diaries/{userId}/{diaryId}   # 日記データ（サブコレクション）
```

### 5.2 User ドキュメント
| フィールド | 型 | 内容 |
|-----------|-----|------|
| id | string | Firebase Auth UID |
| email | string | 登録メールアドレス |
| displayName | string | 表示名（nickname） |
| createdAt | timestamp | アカウント作成日時 |

### 5.3 Diary ドキュメント
| フィールド | 型 | 必須 | 内容 |
|-----------|-----|------|------|
| id | string | auto | ドキュメントID（自動生成） |
| icon | string | 必須 | 選択した絵文字（Unicode またはコードポイント） |
| title | string | 必須 | 日記のタイトル（最大50文字） |
| body | string | 必須 | 日記の本文（最大5000文字） |
| date | timestamp | 必須 | 日記の日付 |
| imageUrl | string | 任意 | 画像のダウンロードURL（Firebase Storage） |
| createdAt | timestamp | auto | ドキュメント作成日時 |
| updatedAt | timestamp | auto | ドキュメント更新日時 |
| userId | string | auto | 所有者のユーザーID |

### 5.4 Firebase Security Rules（案）
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 各ユーザーは自分の日記のみ読み書き可能
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
      match /diaries/{diaryId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
    }
  }
}
```

## 6. 画面遷移図

```
[認証画面]
   ↓ ログイン
[日記一覧（ホーム）] ←→ [日記検索]
   ↓ 作成ボタン         （ホーム内検索バー or 専用画面）
[日記作成]
   ↓ 保存
[日記一覧（ホーム）]
   ↓ カードタップ
[日記詳細] → [日記編集]
   ↓ 削除           ↓ 保存
[日記一覧（ホーム）] [日記詳細]
```

**ナビゲーション構造**:
- ルート: 認証スタック（ログイン / サインアップ）
- メイン: Bottom Tab Navigator（日記一覧 / 検索 / 設定）
  - 日記一覧タブ内部: Stack Navigator（一覧 → 作成 → 詳細 → 編集）
  - 検索タブ内部: Stack Navigator（検索 → 詳細）
  - 設定タブ: プロフィール編集・ログアウト等

## 7. 非機能要件

### 7.1 パフォーマンス
- 日記一覧の初回表示: 3 秒以内に最初の10件を表示
- 画像のアップロード / ダウンロードはバックグラウンドで実行
- Firestore キャッシュを活用し、オフラインでも一覧を表示可能にする

### 7.2 セキュリティ
- Firebase Security Rules でユーザー単位のアクセス制御
- 画像のアップロードは認証済みユーザーのみ許可
- API キーは環境変数で管理（.env ファイル）
- 認証情報は Firebase SDK が安全に管理

### 7.3 オフライン対応
- Firestore のオフラインデータ永続化を有効化
- オフライン時はローカルキャッシュのデータを表示
- オンライン復帰時に自動同期（競合は最新の更新を優先）
