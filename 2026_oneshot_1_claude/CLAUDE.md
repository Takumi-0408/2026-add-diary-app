# CLAUDE.md

このファイルは Claude Code / Codex がこのリポジトリで作業する際のガイダンスを提供します。

## プロジェクト概要

授業内の遊び企画「OneShot 選手権」用のリポジトリ。AI に **1 つのプロンプト** を送って生成された Web アプリのクオリティを競う。

- お題: [PokeAPI](https://pokeapi.co/) を使った Web アプリ
- ルール: AI へのプロンプトは **1 回きり**
- 評価軸: 完成した Web アプリのクオリティ
- デプロイ: なし（ローカルで動けば OK）

**対象**: 個人開発・単発の競技用プロジェクト

## クイックリファレンス

実装後に追記。

## 指示の優先順位

1. **ユーザーの指示**（最優先） — 会話内での直接的な指示
2. **Skills** — Skill ツール経由で呼び出された場合
3. **CLAUDE.md のデフォルト**（最低） — このファイルに記載されたルール

## ワークフロー

すべての依頼に対し、`wf-00-using-workflows` のルールに従ってスキルを選択・実行する。
例外はユーザーが明示的にスキル不要と指示した場合のみ。

## ブランチ戦略

main ブランチへの直接コミットが可能です。feature branch は任意。

## 設計方針

このプロジェクトは「**AI へのプロンプトは 1 回きり**」というルールに従う単発の競技用プロジェクトです。

| 方針                 | 理由                                           |
| -------------------- | ---------------------------------------------- |
| オーバーヘッド最小化 | 1 プロンプトで完結。プロセスより結果重視       |
| ドキュメントは最小限 | 競技用の使い捨て成果物                         |
| スピード重視         | 1 プロンプトの中で動くものを最大化する         |
| ローカル動作のみ     | デプロイ不要。ブラウザで開ければ OK            |

**チーム開発向けの機能（PR レビュー、Issue 管理、フォーマルなプロセス等）は意図的に省略しています。**

## 禁止事項

- プロジェクトの CLAUDE.md や Skills で定義済みの手順をここに複製することを禁止する。スキルの内容を CLAUDE.md に転記せず、スキル名で参照すること。
- 依頼スコープ外の「ついでに改善」を禁止する。
- 将来の仮想要件に備えたコードを禁止する。

## 作業ログ規約

作業ログは `docs/ai-logs/YYYY-MM-DD_{slug}.md` に保存します。
