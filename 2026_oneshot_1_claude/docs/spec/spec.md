# ぽけNS — 仕様書

## 1. プロダクト概要

ユーザーが **ポケモンになりきって投稿する SNS**。レトロゲーム × 紙のような風合いのワイヤーフレーム調デザイン。

初回起動時、草むらをガサガサさせるとランダムで 5 匹のポケモンが出現。プレイヤーはそのうち 1 匹を選んで自分で名前をつけ、その個体「あいぼう」として SNS に参加する。

- ターゲット: モバイル Web（PC では中央に 360 × 720 のフレーム表示）
- デプロイ: なし（`index.html` をブラウザで開けば動く）

## 2. デザイン原則

1. **「自分」じゃなく「あいぼう」が話す** — UI コピーや投稿はモンスター視点。投稿本文は手書きフォント。
2. **レトロゲームの "間" を借りる** — ピクセル UI ラベル、`[コマンド]` 表記、SE 的なテンポ感。
3. **個体差を讃える** — 同じ種でも見た目・性格にばらつきがある前提。図鑑は「種」、プロフィールは「個体」。
4. **競争より生息地** — フォロワー数競争を煽らない。タイプ／生息地で繋がりを設計する。
5. **くらい中の あかり** — 紙ベージュの落ち着いた地色に、緑 / 珊瑚をピンポイントで光らせる。

## 3. MVP スコープ

### 実装する 6 画面

| # | 画面 | 役割 |
|---|---|---|
| 1 | オンボーディング | タイトル → 草むら → 5 匹遭遇 → 選択・命名（4 ステップ） |
| 2 | タイムライン（縦ストリーム） | NPC + 自分の投稿を時系列表示 |
| 3 | 投稿作成 | テキスト + タグ + 気分 + 任意写真の作成 |
| 4 | 投稿詳細 | 本文 + リアクション + コメント |
| 5 | プロフィール（自分） | あいぼうのプロフィール + 自分の投稿一覧 |
| 6 | 図鑑（ずかん） | 発見済み / 未発見の種別一覧 |

### 実装しない（OneShot のスコープ外）

- DM（メッセージ）
- 探索 / ディスカバー
- 他者プロフィール
- グリッド表示版タイムライン
- 通知センター
- 設定画面

- フレーム上部: 「Chrome バー」（22px、URL 風文字を Press Start 2P 7px で表示。装飾的でクリック不可）
- フレーム下部: 「タブバー」（44px、5 タブ）
- 各画面はフレーム内部で縦方向に flex 配置

### 4.4 タブ構成

タブ配置（左→右）:

1. `[HOME]` タイムライン
2. `[SRCH]` 探索（MVP 非対応、押下時はトースト「準備中」または無反応）
3. `[ + ]` 投稿（タップで投稿作成画面）
4. `[ DM ]` メッセ（MVP 非対応）
5. `[ ME ]` じぶん（プロフィール）

タブのアイコン部分は Press Start 2P で `[HOME]` などのテキストを表示する（ピクセルゲームのコマンド風）。

「図鑑」へは プロフィール画面の「図鑑を見る」ボタンから遷移。

### 4.5 コンポーネント仕様

すべてのコンポーネントは描き方を文章で指定する。実装方法（クラス名 / 構造）は実装側で決めて良い。

#### 投稿カード (PostCard)

縦並びの要素:
- ヘッダー行（横並び）
  - 左: アバター（32〜36px 円形、Ink Soft ダッシュ枠 → mid モードでは Ink ソリッド枠 + Grass Soft 背景）
  - 中: `@username`（DotGothic16 11px 太字）+ タイプバッジ
  - 中の下行: `種族名 · 12分前` 風メタ（DotGothic16 9px、Ink Soft）
  - 右: `···` メニュー（Press Start 2P 8px）
- 本文（Yomogi 12〜13px、行高 1.5、1〜3 行程度）
- 任意: 写真プレースホルダ（4:3、内側に斜めストライプパターン）
- リアクション行: `♡ 24`, `💬 5`, `↻ 2`（DotGothic16 10px、Ink Soft）

枠: ダッシュ 1.5px Ink Soft、角丸 6px、内側パディング 10px。

#### タイプバッジ (Pill)

- 形: 横長ピル、角丸 999px
- 枠: 1px ダッシュ（基本）または 1px ソリッド（mid モード）
- カラーマッピング:
  - くさ → Grass
  - ほのお → Coral
  - みず → Sky
  - でんき → Sun
  - その他のタイプ → Ink Soft（または別途定義）
- フォント: DotGothic16 9px、パディング 2px × 7px
- バリアント: 通常（透明背景 + タイプ色文字 + タイプ色枠）／塗り（mid のとき、タイプ色背景 + 白文字）

#### アバター (Avatar)

- 形: 円形
- 枠: 1.25px ダッシュ Ink Soft（mid モードで 1.25px ソリッド Ink + Grass Soft 背景）
- サイズ: small 24px / medium 36px / large 72px
- 内側: PokeAPI のスプライト画像（`object-fit: contain`）。読み込み中は中央に `AVA` のような小さなラベル。

#### ボタン (Button)

- Primary: Grass 背景 + 白文字、角丸 6px、Press Start 2P 9px、パディング 8px × 14px、`[ START ]` のように半角空白で挟む
- Secondary: 透明背景 + Ink 枠（lo はダッシュ、mid はソリッド）、文字は Ink
- 全幅: `block` 修飾子で幅 100%

#### セグメントタブ (Segment)

- 横並びの 3〜5 アイテム、コンテナはダッシュ枠 1.25px、角丸 6px
- アクティブ項目は Ink 背景 + Paper 文字
- 非アクティブは Ink Soft 文字
- フォント: Press Start 2P 8px、各セル間の区切りはダッシュ 1px

#### グラスタイル (GrassTile)

草むらを表すパターン背景:
- ナナメ縞 × 2 方向（45deg と 135deg）の repeating-linear-gradient
- 色: 透明と rgba(91,138,58,.15) の組（lo） または Grass Soft と #b8d29c の組（mid）
- 周囲枠: Grass 色のダッシュまたはソリッド 1.25px
- 角丸 10px 程度

#### 注釈 (Note / Anno)

- Yomogi フォント 11px、Coral 色
- 絶対位置でデザイン上の補足を貼る用途
- 注: 本実装では「↳ タップで揺らせる」など最小限の指示文として残す or 省略可

## 5. 画面仕様

### 5.1 オンボーディング

初回起動時のみ表示する 4 ステップ。`localStorage` に `pokens:partner` が無いときオンボーディングへ。

#### A. タイトル

- 画面: フレーム中央寄せ
- ロゴ: 「ぽけ / NS」を Press Start 2P 18px、2 行表示
- サブコピー: 「〜なりきり SNS〜」を Yomogi 13px、Ink Soft
- 中央: 「HERO ART（草むら + 看板）」のプレースホルダ（160 × 160 の ph 背景）
- 下: 「[ START ]」プライマリボタン（全幅）
- 最下: 「ログイン / 利用規約」のテキスト風リンク（Ink Soft 10px、押下不要）

#### B. 草むら（ガサガサ）

- AppBar: 左 `[ENCOUNTER 1/3]`、右に `SKIP>`（押下で 5 匹遭遇画面へ）
- 説明: 「くさむらが **ガサガサ** している…」（Kaisei Decol 14px、強調部 Coral）
- メイン: グラスタイル領域（フレーム高さの 50% 以上を占有）
  - 中央に `!?` を Press Start 2P 24px
  - 周囲に「ガサッ」「ガサガサ」「ザワッ」の Coral 色注釈
- 補助テキスト: 「↓ 草むらをタップ」（DotGothic16 11px、Ink Soft）
- 最下: 「くさむらを ガサる」プライマリボタン
- 演出: タイル領域をタップすると 0.3〜0.5 秒の左右揺れアニメ。3 回タップ or ボタン押下で次画面へ遷移する間に「ガサガサ…」効果（短時間でよい）

#### C. 5 匹出現

- AppBar: 左 `[ENCOUNTER 2/3]`、右に `SKIP>`
- ヘッドコピー: 「**5ひき** あらわれた!」（強調 Coral）
- 5 匹のカード:
  - 2 × 2 のグリッド + 1 行の横長カード（合計 5 枚）
  - 各カードに、PokeAPI スプライト（64 × 64 ph）、種族名は **初期状態では「???」**、タイプバッジ
  - カードをタップすると下部の「えらぶ」ボタンが有効化、選択中はカードがハイライト
- 補助: 「1ぴき えらんでね」（Ink Soft 10px）
- 最下: 「[ えらぶ ]」プライマリボタン（未選択時は disabled スタイル）
- ロード中: 「ガサガサ…」と書かれたスケルトン or プレースホルダ

#### D. 選択・命名

- AppBar: 左 `もどる<`（前画面へ）、右 `[ENCOUNTER 3/3]`
- ヘッドコピー: 「きみの あいぼう、けってい!」（Kaisei Decol 13px）
- メインカード（グラスタイル背景、角丸）:
  - 中央に選択したポケモンのスプライト（120 × 120）
  - 種族名（PokeAPI の日本語名）を Press Start 2P 11px で表示
  - タイプバッジ（複数タイプならピル並列、性格や形容詞をフェイクで追加してもよい例: 「くさ / かしこい」）
- 入力エリア: 「なまえを つけよう」見出し + テキスト入力（Yomogi 13px、プレースホルダ「ぽち_」）
  - 補足: 「※あとから変更可」
- ボタン行: 左「べつのも」セカンダリ（前画面に戻る）、右「[ はじめる ]」プライマリ（全幅 2:1 比率）

## 6. 技術スタック

- **形態**: 単一の `index.html`（CSS は `<style>`、JS は `<script>` で内包）
- **JS**: Vanilla JavaScript（フレームワーク・ライブラリなし）、ES2020+ 文法
- **ビルド**: なし。`index.html` をダブルクリックでブラウザで開いて動く
- **外部読み込み**:
  - Google Fonts（DotGothic16 / Press Start 2P / Yomogi / Kaisei Decol）
  - PokeAPI（`https://pokeapi.co/api/v2/`）
- **ローカル永続化**: `localStorage`
- **ルーティング**: ハッシュベース（`#/timeline`, `#/post/new`, `#/post/:id`, `#/profile`, `#/zukan`, `#/onboarding`）または単一 SPA でビュー切替（実装側で選択可）
- **状態管理**: ライブラリ不使用。素朴な module-scoped オブジェクト + render 関数で十分
- **テスト**: 自動テストなし（OneShot のため目視確認）

## 7. PokeAPI 連携

### 7.1 使用エンドポイント

| エンドポイント | 用途 |
|---|---|
| `GET /pokemon/{id}` | 個体取得（スプライト URL、タイプ、英名 ID） |
| `GET /pokemon-species/{id}` | 日本語名取得（`names[]` の中で `language.name === 'ja-Hrkt'` のもの） |

### 7.2 スプライト

- 基本: `sprites.front_default`（96 × 96 PNG、ドット絵）
- 高画質代替: `sprites.other['official-artwork'].front_default`（要なら）
- 本プロダクトのワイヤー世界観には **`front_default` のドット絵** を推奨

### 7.3 タイプの日本語マッピング

API ラウンドトリップを避けるため、フロント側に静的辞書を持つ。

| bug | むし | Grass |
| normal | ノーマル | Ink Soft |
| flying | ひこう | Sky |
| poison | どく | Pink |
| ground | じめん | Sun |
| rock | いわ | Ink Soft |

### 7.4 キャッシュ戦略

- 取得した個体データ（個体 + species 統合済み）は `localStorage` に `pokens:poke_cache:{id}` で保存
- 起動時はキャッシュ優先、なければ fetch
- TTL なし（壊れない限り使い続ける）
- ローカル動作優先のため、起動時のフェッチ数を最小化

### 7.5 草むらガサガサ → 5 匹抽選ロジック

- ID 範囲: **1〜151（初代）**（軽量・知名度バランス）
- 重複なしで 5 個を `Math.random()` で抽選
- 各 ID について `pokemon` + `pokemon-species` を **並列フェッチ**（`Promise.all`）
- 各レスポンスから抽出する項目: id / sprite URL / types[] / 日本語名

### 7.6 エラーハンドリング

- フェッチ失敗時: キャッシュフォールバック → それも無ければ「もういちど くさをガサる」リトライ UI を表示
- 個別の失敗（5 匹中 1 匹だけ失敗）: その 1 匹を除外して 4 匹で進めるか、もう 1 匹追加抽選するかは実装判断（簡易には除外で OK）

## 8. データモデル（localStorage）

### 8.1 キー一覧

| キー | 型 | 内容 |
|---|---|---|
| `pokens:partner` | object | 選択したあいぼうの情報 |
| `pokens:posts` | array | 自分の投稿の配列 |
| `pokens:zukan_seen` | array | 発見済みポケモン ID 配列 |
| `pokens:poke_cache:{id}` | object | PokeAPI の個体データキャッシュ |

### 8.2 partner オブジェクトの形

含めるフィールド（自由形式 JSON）:
- ポケモン ID（数値）
- ニックネーム（ユーザー入力、`@username` の元にもなる）
- 表示用の `@username`（ニックネームをローマ字小文字 + アンダースコア風に整形 or 単にニックネームをそのまま使う）
- 種族名（日本語、PokeAPI 由来）
- タイプ配列（日本語表記）
- スプライト URL
- 確定時刻

### 8.3 post オブジェクトの形

- 投稿 ID（UUID 文字列または `Date.now()` ベース）
- 本文（最大 140 字）
- タグ配列（任意）
- 気分（"ごきげん" | "ねむい" | "おこ"）
- 写真 URL 配列（MVP は空配列でも可）
- 作成時刻（ISO 文字列 or epoch）
- 著者: "self" 固定（NPC と区別）

### 8.4 初回起動判定

- `pokens:partner` が無い → オンボーディング画面へ
- ある → タイムライン画面へ

## 9. NPC データ（ハードコード）

実装時に `index.html` 内に JS リテラルとして同梱する。

### 9.1 NPC ポケモン一覧（最低限）

タイプが偏らないよう、以下を最低限含める（8〜12 体推奨）:

| ポケモン ID | 日本語名 | 想定ニックネーム例 | 性格・コピー方向 |
|---|---|---|---|
| 1 | フシギダネ | はっぱくん | くさむら探検家、植物観察 |
| 4 | ヒトカゲ | ヒノタロウ | しっぽ自慢、暑がり |
| 7 | ゼニガメ | アオプヨ | みずたまり評論家、のんびり |
| 25 | ピカチュウ | ピカ介 | でんき気分屋、ピリピリ |
| 39 | プリン | ぷりぷり | おうたとおひるね |
| 54 | コダック | コダ | あたまいたい |
| 63 | ケーシィ | ケシ | テレパシー実況 |
| 92 | ゴース | おばけぱぱ | くらい中の見守り |
| 113 | ラッキー | ハピ | みんなを応援 |
| 133 | イーブイ | エヴィ | きょうの きぶんで タイプ変化 |

上記から **8 体以上** をピックして使う（10 体推奨）。

### 9.2 NPC 投稿テンプレ

各 NPC が **1〜3 件** の投稿、合計 **15〜25 件**。

投稿例（仕様レベル、実装側で言い回しは適宜）:
- くさ系: 「くさむらで ひるね。スヤァ…」「は っぱ つかれた」「ひの ひかりが きもち〜」
- ほのお系: 「しっぽが もえてる!!!」「きょうの くさむらは とくに あつい。」
- みず系: 「みずたまり みつけた。きもちぃ」「ザバァ」
- でんき系: 「ピリピリ きもち〜」「しっぽから ばちっ」
- ノーマル / その他: 「うとうと…」「きょうも たいくつ」「みんな げんき?」

各投稿に持たせるフィールド:
- 著者の NPC ポケモン ID
- 本文（1〜3 行）
- タグ配列（`#ひるね` `#くさむら` `#しっぽ` 等から 0〜2 個）
- 気分（"ごきげん" | "ねむい" | "おこ"）
- 任意の写真有無フラグ（true のとき ph で代用）
- いいね数（10〜120 のランダム固定値）
- コメント数
- リツイート風カウント
- 作成相対時刻表示用文字列（"12分前", "1時間前", "きのう" など、固定でよい）

### 9.3 NPC コメント

各 NPC 投稿に対して **2〜4 件** のコメントを用意する。

各コメント:
- 著者 NPC ポケモン ID（別の NPC）
- 本文（短い、Yomogi で表示）

## 10. アニメーション / 演出

MVP として最低限実装する演出:

| 場面 | 演出 |
|---|---|
| 草むらタップ | グラスタイル領域の左右揺れアニメ（0.3〜0.5 秒） |
| 5 匹出現遷移 | フェードイン or 上から降りる軽い transition |
| 投稿の追加 | 新規投稿カードがフェード or スライドインで現れる |
| ボタン押下 | 軽い透明度変化（0.95 → 1.0 など） |

リッチな演出（パーティクル、SE 等）は MVP スコープ外。時間が余れば任意。

## 11. Lo-Fi / Mid-Fi モード

設計の元ネタには `body[data-fi="lo" | "mid"]` 切替機能があるが、**MVP は `mid` 固定** で十分。`lo`（ダッシュ枠線・透明背景）モードの切替トグルは実装時間が余ったときの拡張。

## 12. 著作権・ルール上の注意

- 本リポジトリは授業内の競技用（個人開発・単発）。**デプロイなし・ローカル動作のみ。**
- PokeAPI の公式名・スプライト・タイプを表示することは、PokeAPI の利用規約に従う限り問題なし
- アプリ自体のブランド名は **オリジナル**（`ぽけNS`）。ポケモンの商標と混同させない
- 本番リリース・配布をする場合は別途検討が必要

## 13. 完成判定（観点）

実装セッション後、以下が満たされていれば成功:

1. `index.html` をブラウザで開くと、初回はオンボーディング、2 回目以降はタイムラインから始まる
2. オンボーディングで草むら → 5 匹出現 → 1 匹選択・命名 が一通りできる
3. タイムラインに NPC 投稿 + 自分の投稿が混じって表示される
4. 「投稿作成」から自分の投稿を追加でき、その投稿が即タイムラインに現れる
5. 投稿詳細でコメントが表示される
6. プロフィール画面で自分のあいぼう情報と自分の投稿サムネイルが見える
7. 図鑑に「あいぼう + NPC として登場したポケモン」が発見済みとして並ぶ
8. デザイントークン（色・フォント）が仕様どおり
9. PokeAPI からのスプライト画像が読み込まれて表示される
10. リロードしてもあいぼう・自分の投稿が保持される
