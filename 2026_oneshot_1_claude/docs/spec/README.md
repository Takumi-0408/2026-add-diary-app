# ぽけNS — 仕様ドキュメント

OneShot 選手権用「ポケモンになりきり SNS」の仕様一式。

## ファイル構成

| ファイル | 内容 |
|---|---|
| `README.md` | 本ファイル。ドキュメント全体の道しるべ |
| `spec.md` | 仕様本体（コンセプト・MVP スコープ・デザインシステム・画面・技術選定・PokeAPI・データモデル・NPC データ仕様） |
| `prompt.md` | 1 ショット実装セッションに投入するプロンプト |
| `../plans/2026-05-14-pokens-subagent-implementation.md` | サブエージェント駆動の実装オーケストレーション計画 |

## OneShot 運用フロー

1. **本セッション（このセッション）**: ドキュメント整備（仕様・設計・スコープ確定 + 実装計画）
2. **実装セッション（別セッション）**: `prompt.md` の内容を 1 度だけプロンプトとして送る。`spec.md`（正本仕様）と `../plans/2026-05-14-pokens-subagent-implementation.md`（実装計画）をコンテキスト資料として併せて渡す
3. ローカルブラウザで `index.html` を開いて動作確認

## 同梱してよいもの / NG なもの

OneShot ルール上、AI へのプロンプトは 1 回きり。コンテキスト同梱はドキュメントのみ許容。

- ✅ `spec.md` / `prompt.md` / `plans/*.md` などの Markdown 仕様
- ✅ 色・フォントの hex / ファミリー名一覧
- ✅ コンポーネントの構造記述（文章・表）
- ❌ HTML / CSS / JS のソースコード断片
- ❌ 既存ワイヤーフレームの生コード
